Week 7:

Project Overview:

This project is a combination of application development and regression modeling construction. Using publicly available data from the Center for Disease control the application will take input from users and run them through the model to determine their predicted success rate based on the chosen parameters. Specifically, this application is based on Artificial Reproductive Technology and is meant to provide user guidance for those in the contemplative stage of seeking medical advice. This tool looks to provide useful information based on the users characteristics such as: estimated cost, success rate stratification on a categorical scale, and expected number of attempts to achieve a live-birth. From the frontend to the backend and all analytical logic is provided by the author.

Goals:

Frontend Design:

Login page: users should be able to register and login to store results

Dashboard: users should be able to input their attributes based on the available drop-down values and receive a processed return value

Graph: graphical trends for different attributes with users able to input different attributes to produce different trendlines

Backend Design:

Database: using SQL user information should be stored and able to be queried

Analysis: Exploratory data analysis performed on the publicly available data and regression model linked to the frontend to perform live analysis

Accomplishments so far:

Login page and database configured with test values and appropriately redirects to the dashboard upon successful login

Database is stored as a local file for now as appropriate for the proxy environment

Dashboard page is appropriately linked with the python data analysis code

Initial and intermediate exploratory data analysis has been completed

Established environment files for streamlined launching of the application

These accomplishments have established the flow of data for this project and serve as the foundation for some more advanced undertakings. Now that there is a minimally working product more advanced portions can be developed.

To do:

Week 11:

What did you do last week?

Last week I explored different techniques to attempt to reduce my RMSE, I tried testing the interaction between two features and if their relationship would help the predictive ability of my model. I applied a logit transformation on the target variable to map the bounded success rate into an unbounded space. Training on the transformed values and mapping them back using the inverse sigmoid function did not improve RMSE. Both proved fruitless in the sense that my RMSE actually either got worse or remained unchanged, however this was actually a very insightful result. This meets the intent of my project well in terms of exploring linear regression and using some statistical analysis gave me some interesting insights. I will likely try to simply add one more attribute before tightening up the project and getting the write up together. The implementation of the attribute combination and logit transformation was time consuming as well as recording the results along the way.
